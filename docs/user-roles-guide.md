# 사용자 역할별 가이드

이 시스템은 **두 가지 인터페이스**를 제공합니다. 사용자 역할에 따라 필요한 것이 다릅니다.

## 👥 사용자 유형별 가이드

### 1. 일반 사용자 (운영자, 프로듀서 등)

**필요한 것:**
- ✅ Slack 앱 설치 (이미 설치되어 있음)
- ❌ Claude Desktop 설치 불필요
- ❌ MCP 설정 불필요

**사용 방법:**
```
/tencent              # 대시보드 열기
/tencent list KBO     # 채널 검색
/tencent schedule     # 스케줄 관리
```

**할 수 있는 것:**
- ✅ 채널 목록 조회
- ✅ 채널 상태 확인
- ✅ 채널 시작/중지/재시작
- ✅ 스케줄 관리 (추가/수정/삭제)
- ✅ 통합 제어 (부모+자식 함께 제어)

**결론:** Slack만 있으면 됩니다! 🎉

---

### 2. 개발자/시스템 관리자

**필요한 것:**
- ✅ Slack 앱 (일반 사용자와 동일)
- ✅ MCP 설정 (선택사항 - 개발/자동화용)

**MCP를 사용하는 이유:**
- 코드에서 자동으로 상태 확인
- 스크립트로 자동화
- AI와 함께 개발 작업

**MCP 설정 방법:**
- **Cursor 사용 (추천):** 별도 설정 불필요, 자동 인식
- **Claude Desktop 사용:** 별도 설치 및 설정 필요

**결론:** 
- 일반 작업: Slack만 사용
- 개발/자동화: MCP 추가 설정 (선택사항)

## 🎯 권장 사항

### 일반 사용자에게 안내할 내용

```
이 시스템은 Slack에서 사용하시면 됩니다.

1. Slack에서 /tencent 명령어 입력
2. 대시보드가 열림
3. 버튼으로 채널 제어

Claude Desktop이나 다른 프로그램 설치할 필요 없습니다!
```

### 개발자에게 안내할 내용

```
Slack Bot으로 일반 작업을 하고,
개발/자동화가 필요하면 MCP를 설정하세요.

- Cursor 사용: 자동 인식 (추천)
- Claude Desktop: 별도 설치 필요
```

## ❓ 자주 묻는 질문

### Q: 모든 사용자가 Claude Desktop을 설치해야 하나요?

**A: 아니요!** 
- 일반 사용자는 Slack만 사용하면 됩니다
- Claude Desktop은 개발자/자동화가 필요한 경우에만 선택사항입니다

### Q: 일반 사용자도 MCP를 사용할 수 있나요?

**A: 필요 없습니다.**
- Slack Bot으로 모든 기능을 사용할 수 있습니다
- MCP는 개발/자동화용 도구입니다

### Q: Slack과 MCP의 차이는?

**A:**
- **Slack Bot**: 모든 사용자가 사용, 제어 가능 (시작/중지 등)
- **MCP**: 개발자용, 조회만 가능 (상태 확인 등)

### Q: 어떤 사용자가 MCP를 사용해야 하나요?

**A:**
- 코드로 자동화가 필요한 개발자
- 스크립트로 상태를 확인해야 하는 운영자
- AI와 함께 개발 작업을 하는 사람

**일반 운영자는 Slack만 사용하면 충분합니다!**
